# CTA 降噪建议（第二批次规划）

基于 213-218 审计结果，给出下一批改动建议。

---

## 1. 首页规则

| 策略 | 说明 |
|------|------|
| 保留最多 2 个强 CTA | 首页目前有 ~12 个主 CTA（含专题入口），专题入口 7 个算合理导航，侧栏资料包 2 个 CTA（"立即领取" + "查看全部 28 个资料包"）功能重叠 |
| AI 阿岩助手 / 公开课 / 资源中心互不抢位 | 当前布局已分区（ask-entry / 专题区 / 侧栏），但侧栏"立即领取"和 ResourceCenterTeaser"查看全部资料包"功能接近，建议合并 |
| **具体建议** | 侧栏"立即领取"弱化为次要入口；ResourceCenterTeaser 保留为主资料入口 |

## 2. /ask/ 规则

| 策略 | 说明 |
|------|------|
| 保持文章优先 | 当前已做到：结果先展示文章，配套资源折叠 ✅ |
| 配套资源折叠 | 已实现折叠 ✅ |
| /survey/ 只做辅助入口 | 已作为折叠内的小字链接 ✅ |
| 不出现强微信 CTA | 当前 wechatHook 在折叠内容内，不算强 CTA ✅ |
| **结论** | /ask/ 已达标，无需改动 |

## 3. /survey/ 规则

| 策略 | 说明 |
|------|------|
| 主 CTA 保留"查看资料详情" | 当前结果页的"查看资料详情 →"为主 CTA ✅ |
| 不要再出现多个"领取" | 当前无"领取"字样 ✅ |
| 微信只作为后续说明 | 当前无微信 CTA ✅ |
| **结论** | /survey/ 已达标，无需改动 |

## 4. /resources/ 规则

| 策略 | 说明 |
|------|------|
| 资源卡片保留"查看资料详情" | 当前有 ✅，但每张卡片同时有"查看文章"造成 CTA 拥挤 |
| 顶部或底部只保留一个"做 30 秒诊断" | Hero 区当前有"做资料诊断" ✅ |
| **具体建议** | ① 合并"查看文章"和"查看资料详情"为主次关系；② 微信关键词改为折叠展示；③ Hero 区"做资料诊断"移到页面底部作为补充入口 |

## 5. 资源详情页规则

| 策略 | 说明 |
|------|------|
| 每页最多 1 个主 CTA | 当前：主 CTA"做 30 秒诊断" ✅ |
| 主 CTA 指向 /survey/ | 当前全部指向 /survey/ ✅ |
| 返回资源中心作为次 CTA | 当前有 ✅ |
| **具体建议** | ① 合并"学习路径"和"公开课"区块为一个"相关资料"区块；② 微信提示降级展示 |

## 6. 文章页规则

| 策略 | 说明 |
|------|------|
| NextUp 保留 | ✅ |
| BackToLearningPath 保留 | ✅ |
| RelatedArticles 最多 2 个 | ⚠️ 当前 max={4}，建议改为 max={2} |
| ArticleNavHints 保留但继续弱化或仅在系列文章显示 | ✅ 当前已做到仅在有系列关系时显示内容 |
| 不恢复 ArticleResourceCTA | ✅ 已移除 |
| **具体建议** | ① 移除侧栏 toc-cta "立即领取"区块；② RelatedArticles 从 4 改为 2 |

## 7. 公开课页规则

| 策略 | 说明 |
|------|------|
| 每页最多 1 个主 CTA | 公开课中心页：每张卡片"查看课程详情"+"查看配套资料"= 2 个 CTA ❌ |
| 不重复出现"回到公开课中心"和"查看资料包"多个强按钮 | 当前"查看配套资料"为同强度按钮 ❌ |
| **具体建议** | 公开课卡片保留"查看课程详情"为唯一强按钮，"查看配套资料"改为文字链接 |
