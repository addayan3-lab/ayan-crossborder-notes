## 1. 资料包中心定位

### 1.1 一句话定位

公开课配套资料包下载入口,与 28 篇文章的 `leadMagnet` 字段直接对接,服务"读完文章 → 领走资料 → 进公开课"的转化漏斗。

### 1.2 服务的不是普通下载站

不做的事情:
- ❌ 不做泛 AI 工具下载站
- ❌ 不做免费 PDF 农场
- ❌ 不做"先关注公众号再解锁 10 个 G 资料"的钩子
- ❌ 不做"全套资料打包下载"的打包营销

要做的事情:
- ✅ 公开课配套:每个资料包对应公开课里讲的具体内容
- ✅ 文章承接:每篇文章结尾有微信钩子,钩子 → 资料 → 进群
- ✅ 运营动作:每个资料包指向 1-3 个具体运营动作
- ✅ 持续更新:资料包不只发一次,新文章发表时同步更新对应资料

### 1.3 与 28 篇文章的对应关系

每篇文章 frontmatter 都有 3 个转化字段:
- `publicLessonUse` — 公开课怎么用
- `leadMagnet` — 资料包名称
- `wechatHook` — 微信承接关键词

资源中心 = `leadMagnet` 的总目录 + `wechatHook` 的对照表 + `publicLessonUse` 的公开课场景索引。

## 6. 暂不开发的内容(避免范围蔓延)

| 不做 | 原因 |
|---|---|
| ❌ 资料包打包下载 | 与"持续更新"理念冲突,鼓励用户按需领取 |
| ❌ 用户登录 / 会员体系 | 增加复杂度,与"公开课转化"主线偏离 |
| ❌ 资料包评论 / 评分 | 资料包是文档不是内容,评论意义有限 |
| ❌ AI 自动推荐资料包 | 28 个资料包,用户手动浏览足够;AI 推荐会引入"自动化"幻觉 |
| ❌ 跨语言版本 | 站点主语言是中文,跨境笔记面向中文运营 |
| ❌ 资料包 PDF 预览 | 多数资料包是表格 / SOP,不适合 PDF 预览 |
| ❌ 资料包倒计时 / 限时 | 与"克制、实操"风格冲突 |
| ❌ 资料包付费墙 | 公开课转化是核心,资料包是漏斗前置,不能设付费 |
| ❌ 站内搜索资料包 | 7 个分类已经清晰,搜索增加复杂度 |
| ❌ 资料包分享返利 | 不在主线上,后续如需单独规划 |

## 8. 资料包维护守则

> 给后续维护者(可能是未来的我,也可能是团队同事)。

1. **新增文章时,先建资料包再建文章**
   - 文章的 `leadMagnet` 字段必须指向一个已存在的资料包
   - 资料包名称必须与文章内容直接相关,不空泛
2. **微信关键词保持唯一**
   - 每次新增资料包,在微信关键词总表里登记
   - 关键词长度 2-4 字,易记易输入
3. **资料包命名要具体**
   - ✅ "8 字段关键词整理表"
   - ❌ "关键词资料包"
4. **公开课场景必填**
   - 每个资料包必须在第 3 节公开课场景索引里能查到
   - 没有公开课场景的资料包,不进资源中心
5. **资料包每季度至少更新一次**
   - 工具变化、价格变化、规则变化时立即更新
   - 更新日志放在 `/resources/changelog/`
6. **删资料包前先停微信关键词**
   - 先在企业微信停用关键词,再删资源中心条目,最后删文章
