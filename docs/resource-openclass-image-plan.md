# 资源中心与公开课页面视觉需求清单

## 页面清单

| 页面 | 路由 | 视觉需求 | 建议方案 |
|------|------|---------|---------|
| 资源中心首页 | /resources/ | 页面 banner / 专题入口卡片 | 现有结构良好，只需一个 banner SVG |
| 7 个资源详情页 | /resources/{slug}/ | 详情页图标 / 模板封面 | 统一模板，每页一个资料包封面 SVG |
| 公开课中心 | /open-class/ | 课程卡片 / 页面 banner | 现有卡片结构良好，只需 banner SVG |
| 4 个公开课详情页 | /open-class/{slug}/ | 课程场景图 | 统一模板，每页一个课程视觉 SVG |

---

## 逐一分析

### 1. /resources/

**当前状态**：已有 7 个专题分类区块、卡片式布局、CTA 按钮。视觉上以文字+背景色为主，无 banner 图。

**需求**：页面顶部可加一个简约 banner SVG，增强页面识别度，不需要大图。

**建议**：先做 SVG banner，不做实景照片。

### 2. 7 个资源详情页

**当前状态**：信息卡片（灰底）+ 详情区块，无资料包封面图。

**需求**：
- 不需要大 banner
- 建议在资料名旁边加一个资料包封面小图（200×200px 或类似 SVG）
- 每个资料包共享同一视觉模板风格，以颜色或专题图标区分

**建议**：统一模板，每个资料包一份 cover SVG。目录结构：`public/images/resources/{slug}/cover.svg`

### 3. /open-class/

**当前状态**：8 节课程卡片，文字为主 + 渐变蓝色标签。

**需求**：
- 不需要每节课程配图（太多图会影响加载）
- 建议顶部加一个"公开课" banner SVG
- 卡片本身靠颜色区分即可

**建议**：一个公共 banner SVG + 每节卡片用小图标区分专题。

### 4. 4 个公开课详情页

**当前状态**：蓝色渐变 hero + 纯文本卡片 + 步骤列表。

**需求**：
- hero 区域可加一个课程主题场景 SVG（尺寸 400-600px）
- 不需要实景图或人物图
- 统一模板风格

**建议**：每个详情页 1 个课程场景 SVG，放在 hero 右侧或视觉留白处。目录：`public/images/open-class/{slug}/hero.svg`

---

## 视觉模板方案

### 资源详情页：统一模板
- 文件名：`public/images/resources/{slug}/cover.svg`
- 尺寸：400×400px
- 风格：商务简约，左上角带专题色块标签
- 7 个专题色：规则（灰）、选品（蓝）、关键词（绿）、Listing（紫）、PPC（橙）、Review（红）、AI 工具（靛）

### 公开课详情页：统一模板
- 文件名：`public/images/open-class/{slug}/hero.svg`
- 尺寸：600×400px
- 风格：深蓝渐变底 + 课程主题对应图标
- 数量：4 个（本批）

---

## 是否会影响加载速度

- SVG 文件平均 8-30KB，4 张额外 SVG 不影响加载速度
- 建议 lazy loading 加载封面图
- 不需要预加载

## 是否建议先做 SVG / 信息图

**是**。所有页面视觉需求均建议用 SVG 实现：
1. 体积小（<50KB / 张）
2. 无需 API 调用
3. 可本地直接编写
4. 风格统一可控
5. 不需要 AI 生成或外部图片服务
6. 不涉及版权问题

不做：AI 写实图、摄影图、复杂 3D 渲染图。
