## 二、下一阶段内容定位

### 2.1 定位声明

> **阿岩跨境笔记 = AI 亚马逊运营实操库**

不再只是"亚马逊知识站"或"亚马逊运营博客"。
下一阶段核心交付：**用户来了能直接学的实操方法 + 直接复制的提示词 + 直接拿走的模板。**

### 2.2 用户来了能学什么

| 用户类型 | 能学什么 |
|----------|----------|
| 刚开店的新手 | 关键词从哪来、Listing 怎么写、广告第一周怎么开 |
| 有产品但不出单 | Review 分析、Listing 优化、PPC 诊断 |
| 想提效的老手 | AI 提示词、自动化流程、数据复盘 SOP |
| 想选品的人 | 市场容量判断、竞品拆解、痛点反推 |

### 2.3 用户能直接复制什么
- AI 提示词（复制即用，替换产品信息即可）
- AI 身份提示词（角色 + 目标 + 输入格式 + 输出格式）
- 模板链接（每个模板对应一篇教学文章）

### 2.4 用户能拿走什么模板
- 每个资源包都有对应的教学文章
- 每篇实操文章结尾都有资源包 CTA
- 所有模板统一在 /resources/ 集中展示

### 2.5 文章到转化的自然流程

```
文章正文 → 文末"想直接拿模板？" → /resources/{slug}/
文章正文 → "不确定自己做得对不对？" → /survey/
文章正文 → "问阿岩" → /ask/
文章正文 → "想系统学？" → /open-class/{slug}/
```

### 2.6 为什么暂时不继续增加公开课

1. 当前 8 门公开课覆盖了所有专题的基础导览。
2. 公开课是"听完就过"的内容，而文章是"看完能复用的资产"。
3. 当前阶段需要沉淀可搜索、可引用、可内链的内容资产。
4. 等实操文章数量足够（60+），再基于文章批量做公开课会更高效。

## 四、文章模板

### 4.1 实操文章模板

```yaml
---
title: "实操标题：解决什么问题的具体方法"
description: "SEO 描述，不超过 160 字"
pubDate: "YYYY-MM-DD"
category: "对应专题"
tags: ["专题", "AI 提效", "实操"]
image: "/images/articles/{slug}/cover.svg"
draft: false
topic: keyword|listing|ppc|review|selection|ai-search|tools
stage: 新手|实操|进阶|复盘
intent: 学习|工具|决策|避坑
relatedTopics:
  - article-slug-1
  - article-slug-2
publicLessonUse: "本篇可作为 X 公开课的 X 环节"
leadMagnet: "附 X 模板下载"
wechatHook: "资料领取方式以资料详情页说明为准"
---
```

**正文结构：**

1. **这篇解决什么问题**（开头 100-200 字，用痛点切入）
2. **适合谁**（一句话筛人）
3. **准备工作**（需要哪些资料/工具）
4. **具体操作步骤**（分步骤，每步配截图或示例）
5. **用 AI 怎么做**（提示词 / SOP）
6. **结果判断标准**（怎么才算做好了）
7. **常见错误**（3-5 个）
8. **对应资料包**（链接到 /resources/）
9. **下一篇推荐**（relatedTopics）
10. **CTA 入口**（→ /survey/ → /ask/ → /resources/）

### 4.4 脱敏案例文章模板

```yaml

title: "案例：某 [品类] 卖家如何通过 [方法] 提升 [指标]"

pubDate: "YYYY-MM-DD"

draft: false
topic: keyword|listing|ppc|review|selection
stage: 复盘
intent: 决策
relatedTopics:
  - article-slug-1
---
```

**正文结构：**

1. **案例背景**（产品、类目、时间，所有名称已脱敏）
2. **已脱敏说明**（哪些信息做了混淆处理）
3. **问题现象**（具体表现：不出单 / ACOS 高 / 退货高）
4. **数据或信息输入**（展示了什么数据）
5. **分析过程**（怎么一步步推出来的）
6. **AI 如何辅助**（AI 在分析中起了什么作用）
7. **最终判断**（做了什么决策）
8. **可复用方法**（用户遇到类似问题可以怎么做）
9. **常见误判**（如果反过来想会错在哪）
10. **对应资料包或诊断入口**（→ /resources/ 或 /survey/）
