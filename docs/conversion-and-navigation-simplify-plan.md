# 转化路径与导航降噪规划方案

**版本**: V01 | **日期**: 2026-06-04  
**对应批次**: 191-196（本批只规划，不改代码）  
**目标**: 根据用户反馈，重新规划网站的转化路径、AI 阿岩助手结果页、资料领取路径、首页专题入口、文章系列标题和上一篇/下一篇导航逻辑。

---

## 当前问题总结

1. **AI 阿岩助手太像领资料入口**：助手结果同时展示建议步骤、推荐文章、资料包（含微信关键词）和公开课，资料包放在第三位但带有微信 hook，整体感受偏营销而非帮助。
2. **资料领取路径只有 Tally + 微信**：没有问卷/诊断环节，用户直接被推到外部表单或微信号。
3. **首页缺选品专题入口**：6 个专题卡片没有选品（虽然选了品放在快速导航中）。
4. **文章关联逻辑不够清晰**：顶部有 ArticleNavHints（上一篇/下一篇 + 系列链接），底部有 RelatedArticles + NextUp + BackToLearningPath + ArticleResourceCTA，多层导航和 CTA 叠加。
5. **部分文章标题未系列化**：选品第 2 篇有系列前缀，但第 1 篇和第 3 篇没有；关键词、Listing、PPC、Review 全都没有系列编号。
6. **领取资料按钮过多**：几乎每个页面都有"立即领取"或"领取资料包"按钮。
7. **正文内没有文章链接**：文章中提到其他文章时没有直接链接，用户必须经过导航组件跳转。
8. **部分文章 topic 归属有歧义**：`amazon-ai-tools-review.md`（AI 工具评测）的 topic 是 `ai-search`，但首页放在"工具模板与资料包"区；`amazon-platform-rules-beginner.md`（平台规则）的 topic 是 `tools`，无独立专题归属。

## T191：AI 阿岩助手结果路径规划

### 当前状态

助手匹配结果按以下顺序展示：

1. 专题标签 + 建议语
2. 建议步骤（有序列表）
3. 推荐文章（卡片链接到 `/articles/{slug}/`）← 第一优先级（合理）
4. 推荐资料包（卡片链接到 `/resources/`，含微信 hook）← 第三位但带营销感
5. 推荐公开课（卡片链接到 `/open-class/`）
6. 微信关键词横幅（仅在 topic 的第一个 resource 有 wechatHook 时显示）

5 个场景卡片触发后直接进入匹配结果，无中间过渡。

### 规划调整

#### 核心原则
- **助手就是助手，不是领资料入口**。用户问问题，助手应该先帮用户解决问题，而不是先推资料。
- **每次结果最多 1 个主要转化 CTA**。不堆叠资料包 + 公开课 + 微信三个 CTA。

#### 建议调整

| 当前顺序 | 建议顺序 | 说明 |
|---------|---------|------|
| 专题标签 | ✅ 保留 | 必须有的结果标识 |
| 建议语 | ✅ 保留 | 核心价值输出 |
| 建议步骤 | ✅ 保留 | 核心价值输出 |
| 推荐文章 | ✅ 保留，提升样式 | 第一优先级，用更丰富的卡片 |
| 推荐资料包 | → 移至末尾，弱化为隐形 | 改为右上角"领取资料"小标签，不占主流程 |
| 推荐公开课 | → 移至末尾，弱化 | 改为底部次要链接，不占主流程 |
| 微信关键词 | → 移至末尾 | 只在资料包显示时附带，不作为独立模块 |
| **主要 CTA** | **新增** | 每次结果最多显示 1 个主要转化 CTA |

#### 具体实施

1. **步骤一: 结果头部（不变）**
   - 专题标签 + 建议语 + 建议步骤

2. **步骤二: 推荐文章（升级）**
   - 保留 2-3 篇文章卡片
   - 去掉"推荐模板"和"推荐公开课"两个独立节
   - 每篇文章卡片可附带"配套资料"角标（极小）

3. **步骤三: 资料/公开课/微信（降噪合并）**
   - 合并为一个"延伸资源"区域，默认折叠或用灰色小字
   - 资料包链接到对应对应资源详情页（不是 `/resources/`）
   - 微信 hook 仅当用户主动展开时才显示

4. **步骤四: 主要 CTA（新增，最多 1 个）**
   - 助手结果底部显示 1 个 CTA，根据场景选择
   - 可选 CTA 类型：
     - 需要入门引导 → "查看学习路径 →"
     - 需要实操 → "查看完整教程 →"
     - 需要资料 → "领取配套资料 →"（只在这个时机出现）
   - 不要在结果里同时出现资料 CTA 和微信 CTA

5. **场景卡片触发后增加过渡**
   - 点击场景卡片 → 显示"正在分析你的问题…"（简短）→ 显示结果
   - 让交互更有"帮用户分析"的感觉，而不是直接跳转到预设结果

#### 不采纳的方案
- ❌ 去掉资料包/公开课（用户仍然需要这些资源）
- ❌ 所有结果都展示"领取资料"（过度营销化）

## T192：资料领取路径规划

### 当前状态

```
资源中心 → 资料详情页 → /lead/ → Tally 外部表单 + 微信 fallback
```

- 资源中心: 28 个资源卡片，每条显示"回复关键词xxx领取"（静态文字）
- 资料详情页: 底部"回复关键词xxx领取" + "立即领取" → `/lead/`
- `/lead/`: "打开领取表单" → Tally 表单 + WeChat ID fallback
- 微信号是用户领资料的必经之路（表单提交后也是加微信）

### 规划调整

#### 核心原则
- **先了解用户，再给资料**。用户填了信息后才能领取，而不是先推到微信。
- **微信号作为最后一步，不是第一步**。
- **新增自定义问卷页**，替代/补充外部 Tally 表单。

#### 建议新增：问卷入口页 `/survey/`

```
资源中心 → 资料详情页 → /survey/（问卷） → 资料下载/微信
```

**问卷字段设计**（可选字段打 *）：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 身份 * | 单选 | 是 | 运营/创业者/服务商/其他 |
| 阶段 * | 单选 | 是 | 开店前/新品期/成长期/成熟期 |
| 问题类型 * | 多选 | 是 | 选品/关键词/Listing/广告/Review/工具/AI |
| 是否已有产品 | 单选 | 否 | 有/没有/正在选品 |
| 联系方式 | 输入 | 否 | 填写后可直接发资料，不强制微信 |

**问卷设计目标**：
- 用户填完问卷后，系统可以推荐更精准的资料包
- 用户填信息后获得资料（即时可见），不是先加微信
- 微信号作为"需要进一步帮助"的可选步骤

**不取代现有路径**：
- /survey/ 是新的补充路径，不是替代
- Tally 表单和 WeChat 继续保留作为 fallback
- 资源中心可以直接跳转到资料详情页，也可以跳转到 /survey/

#### 减少"领取资料"按钮

当前每篇文章底部有：
- ArticleResourceCTA
- 底部大型 CTA（"想把 AI 真正用进亚马逊运营？" → /lead/）

建议减少：
- 去掉 ArticleResourceCTA（文章中的资料包 CTA 过于频繁）
- 保留文章底部 CTA，但最多 1 个
- 资源中心保留 CTA，但简化数量
- 首页资料领取 CTA 从 3+ 减少到 1-2
- 侧边栏资料卡片保留，但弱化"立即领取"按钮

## T193：首页专题入口规划

在现有 6 卡基础上增加第 7 个卡片：**选品专题**。布局调整为：

| 关键词表系列 | Listing 优化系列 |

| 选品专题 | AI 搜索与消费者变化 |

或者简化为 2 列 × 4 行（8 卡）但当前只有 7 个专题，第 8 个留空或改为其他。

更实际的做法：**2 列 × 4 行，第 7 卡为选品，第 8 卡为"查看更多 →"**。

#### 方案 B：重组首页布局


## T195：文章导航降噪规划

### 当前状态

每篇文章页面从上到下的导航组件顺序：

| 位置 | 组件 | 内容 | 建议 |
|------|------|------|------|
| 顶部（标题下方） | ArticleNavHints | 上一篇/下一篇箭头 + 系列文章链接 | ⚠️ 弱化 |
| 底部 | RelatedArticles | 4 篇推荐文章卡片 | ⚠️ 弱化 |
| 底部 | NextUp | 深色"下一篇该读什么"卡片 | ✅ 保留加强 |
| 底部 | BackToLearningPath | 蓝色"专题学习路径"CTA | ✅ 保留 |
| 底部 | ArticleResourceCTA | 暖色"领取配套资料包"CTA | ❌ 移除 |
| 底部（固定） | "#" 想把 AI 真正用进亚马逊运营？ → /lead/ | ❌ 精减 |

**问题**：6 个导航相关区块，每个都是独立视觉模块，用户阅读完正文后面对大量 CTA 和推荐，信息过载。

### 规划调整

#### 最终组件布局（从上到下）

```
正文内容
│
├── 第 1 行：正文内文章链接（正文中首次提到其他文章时嵌入链接，不单独成区块）
│
├── ← 推荐下一篇 →
│   （NextUp 升级：保留现有深色卡片设计，简单直接）
│
├── ← 学习路径 →
│   （BackToLearningPath 保留：蓝色路径 CTA，唯一保留的 CTA）
│
├── ← 可能还需要 →
│   （RelatedArticles 弱化：从 4 卡改为 2 卡紧凑排列，或完全去掉）
│
└── ← 文章底部 CTA（最多 1 个）→
    （将原文底部"想把 AI 真正用进亚马逊运营？" + ArticleResourceCTA 统一为 1 个 CTA）
```

#### 具体调整

| 组件 | 当前 | 建议 | 理由 |
|------|------|------|------|
| ArticleNavHints | 顶部，上一篇/下一篇 + 系列链接 | **移除或移到尾部** | 下一篇导航在底部已有 NextUp，顶部重复；系列链接可通过正文内链接解决 |
| RelatedArticles | 底部，4 卡 | **保留但改为 2 卡，且只在有高度相关文章时显示** | 大多数文章关联度不高，4 卡信息过载 |
| NextUp | 底部，1 卡 | **保留，升级为此核心推荐组件** | 唯一明确的"下一篇该读什么"，最有用 |
| BackToLearningPath | 底部，蓝色 CTA | **保留** | 用户需要知道当前文章在哪个专题中 |
| ArticleResourceCTA | 底部，暖色 CTA | **移除** | 每篇文章都显示"领取资料包"太频繁 |
| 底部大 CTA | 固定位置 | **保留 1 个，合并 ArticleResourceCTA** | 统一为 1 个 CTA，样式低调 |

#### 正文内文章链接

**当前问题**：正文中提到其他文章时没有直接链接，用户必须记住文章名然后手动搜索。

**建议**：在正文中首次提到另一篇文章的地方嵌入链接。

**示例**：
```
AI 做亚马逊市场容量判断（原文中提到）
→ 改为：我们在「选品第 1 篇：AI 估算市场容量」中已经讲过搜索量聚合方法……
→ "选品第 1 篇"加链接到 /articles/ai-market-size-estimate/
```

**操作方式**：直接在 markdown 文件中增加内联链接，不开发新功能。例如：
```markdown
我们在[选品第 1 篇：AI 估算市场容量](/articles/ai-market-size-estimate/)中已经讲过……
```

#### 每篇文章 CTA 数量限制

| 类型 | 最大数量 | 示例 |
|------|---------|------|
| 内容内链接 | 不限 | 正文中引用的文章链接 |
| 推荐下一篇 | 1 | NextUp 组件 |
| 学习路径 | 1 | BackToLearningPath 组件 |
| 转化 CTA | 1 | 底部统一 CTA |
| RelatedArticles | 0-2 | 仅在有高度相关内容时显示 |

总计最多 **4-5 个**导航/CTA 区块（当前约 7-8 个），减少约 40%。
