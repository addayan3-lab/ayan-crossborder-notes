# ayan-conversion-qa

**用途：** 转化路径质量 — 资源领取、问卷、公开课、CTA 密度、微信关键词弱化。不修改业务逻辑和运行代码。

## 前置检查

- [ ] 先读 `AGENTS.md` 第 2 节核心业务原则
- [ ] 先读 `docs/conversion-and-navigation-simplify-plan.md`（如存在）
- [ ] 明确检查范围（页面级、组件级、全站级）

## 规则

### 资源领取

- 资源领取优先走问卷或资料说明页，不要默认跳微信
- 检查 `/resources/` 各资源详情页的 CTA 是否提供问卷/页面领取入口
- 不把资源入口全部改成微信页

### AI 阿岩助手推荐顺序

- 先推荐解释型文章
- 再推荐资源模板
- 最后才推荐问卷、公开课或微信承接
- 不默认跳资源页

### 首页 & 公开课

- 首页必须保留清晰选品入口（如快速导航卡片、专题链接）
- 公开课承接文案贴近新手，不要过度关键词专业化

### 修改后检查

- [ ] `/survey/` 问卷路径正常
- [ ] `/resources/` 资料页列表和详情页正常
- [ ] `/open-class/` 公开课页正常
- [ ] `/lead/` 资源领取入口正常
- [ ] `/ask/` AI 阿岩助手页正常
- [ ] 首页选品入口存在且可点击
- [ ] `npm run build` 通过

## 适用任务

- CTA 噪音审计
- 微信文案或微信 Hook 清理
- 资源详情页/公开课页批量一致性检查
- 转化路径回归
