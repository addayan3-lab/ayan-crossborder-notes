# Command: report

**用途：** 固定报告输出入口。每次任务完成后使用。

## 步骤

1. 收集以下信息：
   - 任务名称
   - 执行结论（成功/失败/部分成功）
   - 修改文件清单
   - 具体改动说明
   - 验证结果（build/SEO/images/git）
   - 风险清单
   - 下一步建议
   - 给 GPT 的回填摘要

2. 写入 `reports/<TASK_NAME>_report.md`

## 报告模板

```markdown
# <TASK_NAME>

## 执行结论

成功/失败/部分成功

## 修改文件

- ...

## 具体改动

- ...

## 验证结果

| Check | Result |
|-------|--------|
| build | ... |
| SEO | ... |
| images:check | ... |

## 风险

- ...

## 下一步建议

- ...

## 给 GPT 的回填摘要

...
```
