### 1.2 关键发现

**1. selection 专题严重空缺**
- 3 篇文章全部是"进阶"阶段（ai-market-size-estimate、ai-competitor-matrix、selection-pain-reverse）
- 没有学习路径页
- 没有面向新手的选品入门文章
- 与 AGENTS.md "首页必须保留清晰选品入口" 原则矛盾：有入口但无新手内容承接

**2. AI 分类结构问题**
- ai-search 专题 5 篇文章中有 2 篇偏理论/消费者视角（consumer-ai-search-amazon、amazon-rufus-alexa-shopping）
- 没有 AI 提示词操作类文章（prompt engineering / 提示词模板）
- 对于面向新手的知识站，新手卖家更需要的是"怎么用 AI 工具"而非"AI 技术原理"

**3. stage 比例失衡**
- 进阶 25%（7 篇），对于一个定位"新手卖家"的站点偏高
- 实操 36%（10 篇）合理但偏向工具型（intent: 工具），缺乏决策型和避坑型实操文章
- 新手 39%（11 篇）看似多，但其中 6 篇是学习路径（偏目录性质），真正解决问题的新手文章只有 5 篇

**4. 内容类型单一**
- 100% 为操作指南/方法型文章
- 0 篇案例拆解（无卖家真实故事）
- 0 篇 AI 提示词文章
- 0 篇资源包配套教学文章
- 0 篇"新手从哪开始"引导型文章
- 缺少差异化的内容形式

**5. 资源包与文章的关联断层**
- 资源包详情页已链接到对应公开课（264 修复），但还没有"资源包怎么填"的配套文章
- 用户领到表格但不知道如何填写，需要手把手教学

### 1.3 优势

- 学习路径体系完整（6/7 个专题有路径）
- frontmatter 标注规范，topic/stage/intent 字段覆盖率好

## 5. 适合做成 AI 提示词文章的

以下文章建议采用"提示词模板 + 实操步骤 + 输出示例"的结构：

- B01~B10 全部 10 篇（通用/关键词/五点/Review/广告/竞品/避坑/客服/全流程/入门）
- C01~C08 全部 8 篇（身份扮演类本质上也是提示词的一种）

提示词文章的核心结构：
```
1. 这个提示词解决什么问题
2. 完整提示词（中文，可直接复制）
3. 提示词拆解：哪里是角色、哪里是任务、哪里是约束
4. 输出示例（脱敏截图或文字结果）
5. 常见翻车场景和调整方法
6. 不适合用 AI 的场景
```

不需要每篇都用同样的 prompt 格式，但要保持"复制即用"的实用性。

## 6. 适合做成脱敏案例拆解的

以下文章建议采用"背景→问题→过程→结果→可复用结论"的结构：

- D01~D10 全部 10 篇

案例文章的核心结构：
```
1. 背景：什么样的产品/类目/阶段
2. 问题：当时遇到的核心困惑
3. 做了什么：操作过程（分步骤）
4. 结果：数据和变化（脱敏）
5. 为什么有效：原理分析
6. 这个案例能复制吗：适用条件和限制
7. 配套资源/工具推荐
```

关键原则：
- 数据必须脱敏：不要出现真实 ASIN、品牌名、销售额绝对值，用百分比和相对值
- 场景要真实：不要编造过于理想的案例
- 失败案例和成功案例同样重要：一个"做了分析后决定放弃的产品"比"爆款打造史"更有参考价值

## 7. 不建议现在做的内容

| 内容类型 | 原因 |
|---------|------|
| 多账号/关联相关内容 | 涉及合规风险，且新手暂时不需，过早写可能误导 |
| 站外引流/TikTok/Facebook 广告 | 新手阶段应集中做站内，站外是进阶 |
| 品牌注册/备案流程 | 流程容易变，且各个站点不同 |
| 税务/VAT/商标/专利 | 专业领域，非运营核心 |
| FBA 物流深度内容 | 新手太多变量，写了容易过时 |
| 供应链/工厂对接内容 | 与亚马逊运营关系远 |
| 过于专业的广告数据模型 | 新手看不懂，写了白写 |
| 继续新增公开课 | 265 已明确下一阶段不增加公开课，聚焦内容资产 |
| 大量翻译成英文 | 目前定位中文市场，不需要过早做英文版 |
| 复杂工具/插件推荐 | 新手连后台都没搞明白，推荐太多工具反而迷茫 |
