## 3. 当前问题

### 问题 1（严重）：领取路径死循环

**路径链路：**
```
资源详情页 → /survey/（问卷） → 资源详情页
```

问卷结果页（`survey/index.astro:286`）链接回资源详情页，资源详情页 CTA 又指向问卷——形成死循环。用户永远无法触达实际的资源领取/下载入口。

**实际资源交付仅存在于：**
- `/lead/`（Tally 表单 + 微信备选）——但详情页和问卷结果页均未链接至此
- 首页侧栏直接链接到 `/lead/`

### 问题 2（中）：首页"资料将通过企业微信发送"

**文件：** `src/pages/index.astro:427`
```
<p class="small-note">资料将通过企业微信发送</p>
```

直接告知用户资料走企业微信，违反 AGENTS.md 第 2.4 条"资源领取优先走问卷/资料说明，不要只跳微信"。

### 问题 3（中）：资源中心 Hero 区展示"28 个微信关键词"

**文件：** `src/pages/resources/index.astro:338-341`
```astro
<div class="hero-meta-item">
  <span class="hero-meta-num">28</span>
  <span class="hero-meta-label">个微信关键词</span>
</div>
```

页面顶部突出展示微信关键词数量，给用户"这个资源中心主要靠微信领"的第一印象。

### 问题 4（低）：`wechatKeyword` 数据残留但未使用

**文件：** `src/pages/resources/index.astro` 数据层

每个资源对象包含 `wechatKeyword` 字段（如 `"规则"`、`"容量"`、`"矩阵"`），但在详情页中并未渲染——CSS 类 `.wechat-keyword` 存在但无对应 HTML。属于半完成状态的旧逻辑残留。

### 问题 5（低）：部分公开课页 WeChat 块文案不一致

**对比：**
- 好的：`ai-tools-for-amazon.astro:64` → "资料详情页会说明领取方式。不确定是否适合？先做 30 秒诊断。"
- 待改进的：`operation-review-system.astro:112`、`keyword-to-listing.astro:119` 等 → 传统 `wechat-block` 结构，内容未明确优先走问卷。

### 问题 6（低）：AI 助手规则中的 `wechatHook` 良好

**文件：** `src/data/ayan-assistant-rules.ts`

所有资源的 `wechatHook` 统一为 `"资料领取方式以资料详情页说明为准。"` — 不直接推微信，符合规则。
