# 025｜文章 emoji 与目录修复

## 执行者
OpenCode

## 模式
write

## 项目路径
C:\rpg\ayan-crossborder-notes

## 目标
修复文章《如何用 AI 做亚马逊关键词表》的阅读体验问题：

1. 文章正文可以适当保留少量 emoji
2. 文章目录里不要出现 emoji
3. H2/H3 标题里尽量不要放 emoji，避免目录和字体显示异常
4. emoji 应主要放在提示块、重点句、风险提醒块里
5. 保持整体专业、高级、克制，不要花哨

## 修改范围
优先修改：

- src/pages/articles/[slug].astro
- src/content/posts/ai-keyword-table.md

## 具体要求

### A. 目录去 emoji
如果文章目录是从 headings / h2 / h3 自动生成的，请在渲染目录文字时做清洗：

- 去掉 emoji
- 去掉奇怪方块符号
- 保留中文标题文本
- 保留序号

例如：

“💡 一、先明确：这张关键词表到底要用来干什么”

目录里应显示为：

“一、先明确：这张关键词表到底要用来干什么”

### B. 正文标题不要 emoji
把 ai-keyword-table.md 里的 H2/H3 标题前 emoji 去掉。

错误示例：

## 💡 一、先明确：这张关键词表到底要用来干什么

正确示例：

## 一、先明确：这张关键词表到底要用来干什么

### C. 正文可以保留少量 emoji
emoji 可以放在提示块里，例如：

> 💡 重点：关键词表不是为了收集更多词，而是为了帮你决定哪些词放 Listing，哪些词拿去广告测试。

> ⚠️ 注意：搜索量高不代表适合你的产品。相关性不够，流量越大，广告费浪费越快。

> ✅ 建议：新手先做 50 到 100 个词，每周用广告搜索词报告更新一次。

### D. 图片显示保持正常
不要破坏当前图片自适应效果。

### E. 验收标准
1. 文章目录没有 emoji
2. 正文标题没有 emoji 或方块符号
3. 正文重点块可以有少量 emoji
4. npm run build 通过
5. npm run seo:audit 通过，Fail: 0

## 输出要求
完成后输出：
1. 修改了哪些文件
2. 目录 emoji 如何处理
3. 正文 emoji 如何保留
4. build / seo:audit 结果
